SignalArray

Reactive array management with familiar mutable-style API while maintaining immutability

Overview

SignalArray provides a reactive wrapper around arrays in signals, offering familiar methods like push, remove, and move while automatically maintaining immutability and triggering change detection.

Available Methods

SignalArray provides a comprehensive API for array manipulation:

Method Description
push(...items) Add items to the end
insert(index, item) Insert item at specific index
removeAt(index) Remove item at index
removeBy(predicate) Remove items matching condition
updateAt(index, updater) Update item at index
move(from, to) Move item between indices
set(newArray) Replace entire array
clear() Remove all items

Best Practices

‚ö†Ô∏è Important Notes

  • Always use items.value() with parentheses to access the array
  • Never mutate the underlying signal directly
  • Use track functions in @for loops for optimal performance
  • Properties like length and at() are signals, call them as functions

API Reference

Static Methods

SignalArray.from<Parent, K>(parent: WritableSignal<Parent>, key: K): SignalArray<T>

Creates a SignalArray from a parent signal's array property.

SignalArray.wrap<T>(signal: WritableSignal<T[]>): SignalArray<T>

Wraps a standalone signal containing an array.

Properties

value: Signal<T[]>

Read-only signal exposing the array. Use in templates: items.value()

length: Signal<number>

Reactive signal for array length. Use: items.length()

Methods

at(index: number): Signal<T | undefined>

Returns a signal for the item at the given index.
